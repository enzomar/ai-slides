Chapter 1 sets the foundation. Key message: AI is not magic — it is mathematics + massive data. Slide 01 · Two Paradigms

Deterministic
vs Probabilistic

Deterministic
Same input → Same output. Always.
0 1
  • SQL query → always same rows
  • SHA-256 hash function
  • 2 + 2 = 4 (every time)
  • Sorting algorithm
Probabilistic
Same input → Different possible outputs.
Paris 92% Lyon 5% Rome 2% … 1%
  • ChatGPT answering a question
  • Weather forecast (70% rain)
  • Image generation (DALL·E)
  • Recommendation engine
Key takeaway: LLMs are probabilistic — the same question can produce different, valid answers. This is a feature, not a bug.
Ask the audience: "Who was surprised when the same question to ChatGPT gave a different answer each time?" This is the key mental model shift. Engineers from SQL/API backgrounds struggle most with this — 'non-determinism is a feature, not a bug.' Temperature controls randomness — preview that for Slide 4. Deterministic = guaranteed; probabilistic = optimized for the likely best answer.

Slide 02 · Evolution

A Brief History of
Probabilistic Models

1950s

Markov Chains

Predict next state from current. Foundation of probabilistic models.

1980s

Hidden Markov Models

Speech recognition & NLP. First voice assistants.

1990s

Neural Networks

Brain-inspired. Backpropagation enables learning from data.

2012

Deep Learning

AlexNet wins ImageNet. GPU revolution begins.

2017

Transformers

"Attention is All You Need" — Google Brain.

2020+

Large Language Models

GPT-3/4, Claude, Gemini. Billions of parameters.

Key idea: Each era solved a limitation of the previous one. Transformers unlocked long-range context understanding — the key to human-like language.
The 2017 Transformer paper ('Attention Is All You Need') is the inflection point. Before: RNNs forgot distant context. After: every token sees every other token simultaneously. GPT-3 (2020) proved that scale alone unlocks in-context learning — no fine-tuning needed. Ask: 'What changed most between 2017 and today?' Answer: scale and data volume, not the core architecture.

Interactive · The Heart of Transformers

Self-Attention in Action

Insight: Each row shows which words a token "pays attention to." Brighter = stronger connection. This is how "it" resolves to "cat", not "mat".
Each row = one token querying all others. Brighter = stronger attention. Notice: 'it' attends strongly to 'cat' — the model resolves pronoun references without explicit programming. This is emergent from training on billions of sentences. Multiple heads run in parallel — each learns different linguistic patterns (subject-verb, adjective-noun, long-range coreference). This is why Transformers beat RNNs on understanding.

Slide 04 · Configuration

Key Parameters

Temperature
Controls randomness. Low = focused. High = creative.
0.0balanced2.0
0.2 (code) → 0.7 (chat) → 1.0 (creative writing)
Max Tokens
Max response length. 1 token ≈ ¾ word.
256 short 1k medium 4k long 128k full doc
More tokens = more cost. Balance wisely.
Top-K
Consider only K most likely next words.
K=5 kept
Range: 1 – vocab size · Typical: 40 – 100
Top-P (nucleus)
Pick tokens whose cumulative probability ≤ P.
P=0.1 → very focused
P=0.5 → balanced
P=0.95 → creative
Range: 0.0 – 1.0 · Default: 0.9
Pro tip: For code/extraction → Temperature 0.2. For brainstorming → Temperature 0.8–1.0. Avoid changing Top-K and Top-P at the same time.
Temperature is the most important parameter to understand. 0 = deterministic (always picks the highest-probability token), 1 = creative/varied. For production use cases: code/extraction → 0.1–0.3; brainstorming → 0.8–1.0. Tokens ≠ words: ~1.3 tokens per word in English. Max Tokens = your cost + response-length budget. Avoid changing Top-K and Top-P simultaneously — they interact.

Slide 05 · Infrastructure

Hardware for AI

🖥️

CPU — Central Processing Unit

Good for inference on small models (≤7B params). Sequential tasks, low throughput.

✓ Run: Phi-3, Mistral 7B ✓ Run: LLaMA 3 8B (slow) ✗ Train: not practical
🎮

Consumer GPU (RTX 4090)

24 GB VRAM. Fast inference on medium models. Light fine-tuning with LoRA/QLoRA.

✓ Run: up to 13B (quantized) ~ Fine-tune: LoRA only
⚡

Data Center GPU (A100 / H100)

80 GB VRAM. Run & fine-tune large models. Multi-GPU clusters for training.

✓ Run: 70B+ models ✓ Train: full fine-tuning
☁️

Cloud TPU / GPU Clusters

Thousands of chips. Pre-training frontier models (GPT-4, Claude, Gemini). $10M–$100M+ runs.

✓ Train: frontier models ✓ Run: any model at scale
GPUs are essential because neural net layers are giant matrix multiplications — GPUs run thousands of these in parallel. H100 = 80GB VRAM, enough for 70B parameter models at fp16. Cloud (Azure OpenAI, AWS Bedrock, Google Vertex) removes CapEx — pay per token. Edge/MacBook inference is now viable for 7B–13B models via quantization (llama.cpp, Ollama). The cost curve is dropping ~10× per year.

Tip: There is no single "best" model. Choose based on your use case: cost, latency, context length, privacy requirements, and task complexity.
This landscape changes every 3-6 months. Key decision factors: (1) closed vs open — closed models are more capable but data leaves your infra; (2) context window — Gemini 1M is great for ingesting entire codebases; (3) cost — GPT-4o mini is 20x cheaper than GPT-4o for 80% of tasks; (4) EU sovereignty — Mistral for GDPR-sensitive deployments. Recommend: start with GPT-4o or Claude for quality, then optimize down to smaller/cheaper models.
