Patterns Cheat Sheet

🎭 Role

Assign an expert persona to improve depth and style.

Act as a senior security engineer…

✓ Tone, expertise, vocabulary

🔗 Chain-of-Thought

Force step-by-step reasoning before the final answer.

Think step by step.
First, identify…
Then, calculate…

✓ Math, logic, complex analysis

📋 Few-Shot

Show 2–5 examples to define the expected format and style.

"Great!" → Positive
"Awful." → Negative
"Decent." → ?

✓ Custom formats, niche tasks

📐 Structured Output

Specify exact format: JSON, table, YAML. Enables machine parsing.

Return JSON:
{"name":…,"score":…,"tags":[…]}

✓ APIs, pipelines, automation

⚡ ReAct

Reason → Act → Observe → Repeat. Used in agents with tool access.

🔄 Meta-Prompting

"Improve this prompt…" — let the model optimise its own instructions.

🚫 Negative Constraints

"Do NOT use bullet points." Explicit guardrails improve reliability.

📌 Grounding

"Answer ONLY using the document below." Reduces hallucinations.

Economics · AI Deployment

The Cost of AI

💰 What Drives Cost
📦 Model Size

Larger models cost 10–100× more per call than smaller ones (GPT-4o mini, Haiku).

🔤 Tokens In + Out

Pricing is per token. Long context, verbose prompts all increase cost. Output tokens cost 3–5× more.

🔁 Call Frequency

1M users × 10 msgs/day × 1k tokens = 10B tokens/day → $10k/day at typical pricing.

🗃️ Vector Operations

Embedding + vector search add cost for every RAG query. Caching reduces repeated costs.

🎯 Cost Optimisation Strategies
1. Model routing — use small models for simple tasks

GPT-4o mini for classification, GPT-4o for complex reasoning only.

2. Prompt compression — shorter prompts, same quality

Remove boilerplate, use structured context, define output schema up front.

3. Caching — cache repeated prompts (semantic caching)

GPTCache, Langchain cache: FAQs avoid repeated LLM calls entirely.

4. Fine-tuning — distil GPT-4 behaviour into a 7B model

One-time fine-tuning cost replaces ongoing inference cost for high-volume use cases.

Annex · Local AI

Running LLMs
Locally

Why run locally?
  • 🔒 Privacy — data never leaves your machine
  • 📶 Offline — no internet connection required
  • 💰 Free — no API costs after setup
  • 🔧 Dev-friendly — custom fine-tunes, no rate limits
Quick start — Ollama
# install (macOS / Linux)
brew install ollama
# pull & run a model
ollama run llama3.2
ollama run mistral
# OpenAI-compatible API
ollama serve # → localhost:11434
