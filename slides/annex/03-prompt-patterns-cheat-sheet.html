Prompt Engineering
Patterns Cheat Sheet

🎭 Role

Assign an expert persona to improve depth and style.

Act as a senior security engineer…

✓ Tone, expertise, vocabulary

🔗 Chain-of-Thought

Force step-by-step reasoning before the final answer.

Think step by step.
First, identify…
Then, calculate…

✓ Math, logic, complex analysis

📋 Few-Shot

Show 2–5 examples to define the expected format and style.

"Great!" → Positive
"Awful." → Negative
"Decent." → ?

✓ Custom formats, niche tasks

📐 Structured Output

Specify exact format: JSON, table, YAML. Enables machine parsing.

Return JSON:
{"name":…,"score":…,"tags":[…]}

✓ APIs, pipelines, automation

⚡ ReAct

Reason → Act → Observe → Repeat. Used in agents with tool access.

🔄 Meta-Prompting

"Improve this prompt…" — let the model optimise its own instructions.

🚫 Negative Constraints

"Do NOT use bullet points." Explicit guardrails improve reliability.

📌 Grounding

“Answer ONLY using the document below.” Reduces hallucinations.
