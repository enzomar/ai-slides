The Cost of AI

💰 What Drives Cost
📦 Model Size

Larger models cost 10–100× more per call than smaller ones (GPT-4o mini, Haiku).

🔤 Tokens In + Out

Pricing is per token. Long context, verbose prompts all increase cost. Output tokens cost 3–5× more.

🔁 Call Frequency

1M users × 10 msgs/day × 1k tokens = 10B tokens/day → $10k/day at typical pricing.

🗃️ Vector Operations

Embedding + vector search add cost for every RAG query. Caching reduces repeated costs.

🎯 Cost Optimisation Strategies
1. Model routing — use small models for simple tasks

GPT-4o mini for classification, GPT-4o for complex reasoning only.

2. Prompt compression — shorter prompts, same quality

Remove boilerplate, use structured context, define output schema up front.

3. Caching — cache repeated prompts (semantic caching)

GPTCache, Langchain cache: FAQs avoid repeated LLM calls entirely.

4. Fine-tuning — distil GPT-4 behaviour into a 7B model

One-time fine-tuning cost replaces ongoing inference cost for high-volume use cases.
