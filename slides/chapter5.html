Slide 01 · Concept

What is an AI Agent?

An AI agent is an LLM that can observe its environment, reason, decide, and take actions to reach a goal — in a loop.

LLM vs Agent
Plain LLM
Q → A (one shot)
AI Agent
Goal → Plan → Tools → Result
  1. 1
    Perceive
    Reads input: user message, tool results, memory
  2. 2
    Reason
    Decides what action (or tool) to use next
  3. 3
    Act
    Calls a tool: web search, code exec, API call, file write…
  4. 4
    Observe & Loop
    Checks result, repeats until goal is achieved

Slide 01b · Agentic Patterns (DeepLearning.AI)

Andrew Ng's 4
Agentic Design Patterns

🪞 Reflection
The agent reviews its own output, critiques it, and iterates. Like a writer editing their own draft before submitting.
Example: Code generation → self-review → fix bugs → re-test
🔧 Tool Use
The agent calls external tools — web search, code execution, APIs — to gather information and perform actions beyond text generation.
Example: Research question → search web → read papers → synthesize
📋 Planning
The agent creates an execution plan before acting: breaks complex goals into sub-tasks, orders them, and tracks progress.
Example: "Build a website" → plan → design → code → test → deploy
🤝 Multi-Agent Collaboration
Multiple specialised agents work together: one writes code, another reviews it, a third runs tests. Debate and consensus yield better results.
Example: Writer agent + Critic agent + Editor agent → polished article
Source: Andrew Ng, DeepLearning.AI — "Agentic Design Patterns" series, 2024. These patterns can be combined: an agent can plan, use tools, reflect on results, and collaborate with other agents.

Slide 02 · ReAct

Reason + Act:
The ReAct Loop

Example: "Book me Paris flights for next Tuesday"

Thought: I need to find today's date first.
Action: get_current_date()
Observation: 2026-03-06

Thought: Next Tuesday is 2026-03-10. Search flights.
Action: search_flights("Paris", "2026-03-10")
Observation: 3 results found, cheapest €189

Thought: I should confirm before booking.
Action: ask_user("Confirm cheapest at €189?")
Observation: "Yes, confirm."

Action: book_flight(flight_id="AF1234")
✓ Booking confirmed. PNR: XK7291
🧠 Thought — Internal reasoning step
→
⚡ Action — Call a tool / function
→
👁️ Observation — Tool result returned
→
🔁 Repeat until goal reached or human asked

Slide 03 · Tool Use

Tool Use &
Function Calling

You describe tools as JSON schemas. The model decides when and how to call them.

// Tool definition (schema)
{
  "name": "get_weather",
  "description": "Get current weather",
  "parameters": {
    "city": { "type": "string" }
  }
}

// Model output (function call)
{ "tool": "get_weather",
  "args": { "city": "Paris" } }
Common Tool Categories
  • 🔍 Search — web, vector DB, SQL queries
  • 💻 Code execution — Python sandbox, shell
  • 🌐 APIs — weather, maps, CRM, ERP, Slack
  • 📂 File system — read, write, create, delete
  • 📧 Communication — send email, calendar, Slack
  • 🧪 Testing — run tests, deploy, CI/CD triggers

Slide 05 · Multi-Agent

Multi-Agent
Systems

Complex tasks split across specialised agents that collaborate, verify each other, and hand off work. A supervisor agent orchestrates the flow.

LangGraph LangChain ecosystem production-ready

Models agent workflows as a state machine graph: nodes are LLM calls or tool invocations; edges are conditional transitions. A supervisor node routes tasks to specialist worker nodes.

# Supervisor → Workers pattern
Supervisor ──▶ ResearchAgent
            ──▶ CodeAgent
            ──▶ CriticAgent ──▶ END
✓ Checkpointing & resumability ✓ Human-in-the-loop nodes ✓ Streaming
Orchestration frameworks
LangGraph
State-machine graphs. Best for deterministic, inspectable pipelines. Production standard.
AutoGen
Conversation-based multi-agent chat (Microsoft). Great for debate / review loops.
CrewAI
Role-based crews with YAML config. Fast prototyping but less control at scale.
Recommendation: Start with LangGraph for anything going to production. Use CrewAI for fast demos. AutoGen for peer-review agent patterns.

Chapter 5 · Summary

Key Takeaways

LLM → Agent: the key difference

LLM = single call. Agent = loop: observe → reason → act → observe → repeat until done.

ReAct: reason then act

ReAct: Thought → Action → Observation — a transparent audit trail of every agent decision.

MCP: the USB-C of AI tools

MCP standardises agent-tool connections. Build once, connect anywhere — the USB-C moment for AI.

Agents fail differently than LLMs

Agent failures are real-world: runaway actions, costs, irreversible effects. Always define a kill switch.
