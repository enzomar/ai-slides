Chapter 04

RAG Architecture

ChatGPT knows everything that was on the internet before its training cutoff — but nothing about your internal docs, last week's meeting notes, or your proprietary data. RAG is the bridge that changes that.

RAG Architecture
Slide 01 · The Problem

Why LLMs Need
External Knowledge

LLM Limitations
  • Knowledge frozen at training cutoff
  • No access to private / enterprise data
  • Can hallucinate outdated facts
  • Context window limits large corpora
RAG Solution
  • Retrieve relevant chunks at query time
  • Ground answers in real, citable sources
  • Works on proprietary documents
  • Always up-to-date (live index)
Core idea: Instead of retraining the model, inject relevant documents into the prompt at runtime.

Embeddings &
Vector Databases

What is an Embedding?

A numerical vector that captures the semantic meaning of text.

"The cat sat on the mat"
→ [0.21, -0.84, 0.13, 0.67, …]
// 1536 dimensions (OpenAI ada-002)

Similar sentences → similar vectors (close in space)

Slide 04 · Chunking

Chunking Strategies

Fixed Size

Split every N tokens with a sliding overlap window.

# 512 tok, 10% overlap
"The transformer architecture…"
chunk 1 [0–512]
chunk 2 [460–972] ← overlap

✓ Simple, fast, predictable

✗ May cut sentences mid-thought

Tools
LangChain RecursiveCharacterTextSplitter LlamaIndex TokenTextSplitter tiktoken
Best for: plain text, logs, homogeneous corpora
Semantic

Split on sentence / paragraph / heading boundaries.

# Markdown-aware split
## Section: Pricing
→ chunk boundary ↑
## Section: Returns
→ chunk boundary ↑

✓ Preserves meaning, respects structure

✗ Variable sizes, needs structure in source

Tools
LangChain SemanticChunker LlamaIndex SemanticSplitterNodeParser spaCy sentencizer Unstructured.io
Best for: docs, manuals, markdown, HTML
Hierarchical

Large parent chunks for context + small child chunks for precision.

Parent [2048 tok] ← sent to LLM
├─ Child [128 tok] ← retrieved
├─ Child [128 tok]
└─ Child [128 tok]

✓ High precision retrieval + rich context

✗ More storage, complex pipeline

Tools
LlamaIndex HierarchicalNodeParser LangChain ParentDocumentRetriever LlamaIndex AutoMergingRetriever
Best for: long reports, legal docs, books
Rule of thumb: Start with RecursiveCharacterTextSplitter (chunk 512, overlap 64). Switch to Semantic or Hierarchical when retrieval recall drops below 80%.

Chapter 4 · Summary

Key Takeaways

RAG solves frozen knowledge

LLMs have frozen knowledge. RAG retrieves relevant docs at query time — no retraining needed.

chunk → embed → retrieve → generate

Split → embed → store. At query time: find closest chunks → inject → generate grounded answer.

Chunk size changes everything

Too large: noise. Too small: no context. Sweet spot: 256–512 tokens, 10–20% overlap. Measure recall@k.

RAG beats fine-tuning for facts

Fine-tuning = new skills. RAG = new facts. For enterprise QA, RAG is faster and more explainable.
