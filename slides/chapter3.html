In 2022, writing a REST API from scratch took hours. In 2025, it takes minutes. This chapter shows you exactly how — and how to stay the human who directs the machine, not the one replaced by it.

AI-Assisted Coding
Slide 01 · Concept

What is AI-Assisted Coding?

AI-assisted coding spans a wide spectrum — from intelligent autocomplete to fully conversational development. Karpathy's "vibe-coding" (Feb 2025) captures the most radical end:

  1. 1
    Describe what you want
    In plain language, not code
  2. 2
    AI generates the code
    Full functions, files, even entire projects
  3. 3
    You review & iterate
    Accept, modify, or refine via conversation
The Karpathy Quote
"I just see stuff, say stuff, run stuff, and copy-paste stuff, and it mostly works."
— Andrej Karpathy, Feb 2025
Spectrum: ① Autocomplete (Copilot) → ② Chat-assisted edits (Cursor, Copilot Chat) → ③ Fully conversational "vibe-coding." The skill shifts from writing code to reviewing and directing it.

Slide 04 · Reality Check

When It Shines &
When It Doesn't

✓ Works Great
  • CRUD operations & boilerplate
  • Unit tests & test data generation
  • Regex, SQL, config files
  • Prototyping & MVPs
  • Documentation & comments
⚠ Be Careful
  • Complex business logic (domain-specific)
  • Security-critical code
  • Performance-sensitive algorithms
  • Legacy codebase with poor docs
  • Compliance & regulated systems
Golden rule: Never ship AI-generated code you don't understand. Vibe-coding amplifies your skills — it doesn't replace them.

Slide 06 · Exercise

🛠️ Hands-On Exercise

Challenge: AI Pair Programming

Use an AI coding assistant to build a small utility. Compare the AI-generated code against what you would write manually.

📋 TASK

Build a CLI tool that reads a CSV file and outputs a summary: row count, column names, and the top 5 values of a user-specified column.

⚡ Process
1. Describe the task to Copilot Chat / Cursor / Claude
2. Review the generated code — does it handle errors? Edge cases?
3. Ask the AI to add tests, then run them
4. Refactor: ask for improvements — "Make this production-ready"
Reflection: What did the AI get right on the first try? What required human correction? Would you trust this code without review?

Chapter 3 · Summary

Key Takeaways

AI codes faster, you code smarter

Studies show 35–55% real-world productivity gains. You direct; AI accelerates.

Right tool, right task

Copilot for inline, Cursor for files, Claude/ChatGPT for architecture. Match tool to context size.

AI fails at the edges

Struggles with: unknown codebases, security logic, novel algorithms. You are the senior developer.

The vibe-coding workflow

Spec → generate → iterate → review diff → commit. Your skill is specification and critical review.
