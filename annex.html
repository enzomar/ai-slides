Glossary · A–L

Key Terms (A–L)

Attention
Mechanism that lets the model weigh the relevance of all tokens in context when generating the next token.
Context Window
The maximum number of tokens (input + output) the model can process at once. GPT-4: 128k. Claude 3.7: 200k.
Embedding
A dense numerical vector representing the semantic meaning of text. Similar texts have similar vectors.
Fine-Tuning
Continuing training of a pre-trained model on a smaller, domain-specific dataset to specialise its behaviour.
Hallucination
When a model generates plausible-sounding but factually incorrect or invented content.
Inference
The process of running a trained model to generate output for new inputs. Opposite of training.
LLM
Large Language Model. A neural network trained on massive text corpora to predict and generate text.
LoRA
Low-Rank Adaptation. Efficient fine-tuning that trains fewer parameters via small adapter matrices.

Glossary · M–Z

Key Terms (M–Z)

MCP
Model Context Protocol. Open standard by Anthropic (2024) for connecting AI apps to external tools.
Parameters
The numerical weights of a model learned during training. GPT-4: ~1.8T params. More ≈ more capable.
RAG
Retrieval-Augmented Generation. Retrieves relevant documents at query time and injects them into the prompt.
System Prompt
Hidden instruction block before the user's message. Sets the model's role, tone, and constraints.
Temperature
0 = deterministic / predictable. 1 = creative / variable. Controls randomness in token selection.
Token
The basic unit of text for LLMs. ~¾ of a word on average. 1000 tokens ≈ 750 words.
Transformer
Neural network architecture at the core of all modern LLMs. Introduced by Google (2017).
Vector Database
Database optimised for high-dimensional vector search. Used for semantic search in RAG systems.
